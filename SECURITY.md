# Security Policy

## Supported Versions

En general, damos soporte de seguridad a las ramas activas mantenidas por el
equipo. Si no estas seguro de la version que usas, abre una consulta en los
canales de mantenimiento.

| Version | Soporte de seguridad |
| ------- | -------------------- |
| main    | Si                   |
| develop-* | Mejor esfuerzo      |
| otras ramas antiguas | No       |

## Report a Vulnerability

Si encuentras una vulnerabilidad de seguridad:

1. No la publiques en un issue publico.
2. Reportala por correo a: <security@deasy.dev>
3. Incluye la mayor cantidad de contexto posible:
   - Modulo afectado (backend, frontend, docker, scripts, etc.)
   - Pasos para reproducir
   - Impacto estimado
   - Evidencia tecnica (logs, payloads, stack traces)
   - Propuesta de mitigacion si la tienes

## Disclosure Process

- Confirmaremos recepcion del reporte en un plazo objetivo de 72 horas.
- Investigaremos y clasificaremos la severidad.
- Si aplica, prepararemos fix y coordinaremos divulgacion responsable.
- Agradecemos reportes hechos de buena fe y no tomaremos acciones contra
  investigacion responsable.
